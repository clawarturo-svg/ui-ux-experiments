# UI/UX Review Process

## Objetivo
No dar una landing o web por terminada cuando simplemente "funciona". El sitio solo se considera terminado cuando pasa un loop de revisión visual y UX en desktop y mobile.

## Loop obligatorio
1. Build / edit de la página
2. Abrir en browser real
3. Sacar screenshot desktop
4. Sacar screenshot mobile
5. Revisar:
   - jerarquía visual
   - contraste
   - spacing
   - legibilidad
   - claridad del hero
   - CTA
   - consistencia entre bloques
   - responsive
   - errores visuales obvios
6. Aplicar mejoras
7. Repetir hasta quedar conforme

## Criterios mínimos
- Hero claro y fuerte
- CTA visible
- Tipografía legible en mobile
- Cards no rotas
- No overflow horizontal
- Fondo/animación no debe molestar lectura
- Secciones con ritmo visual consistente

## Regla
Siempre validar en mobile también.
